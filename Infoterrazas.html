SunTrack Madrid

SunTrack Madrid es un proyecto creado para la segunda convocatoria de los Premios a la Reutilización de Datos Abiertos del Ayuntamiento de Madrid. Su objetivo es analizar, modelar y visualizar de forma objetiva la exposición solar de las terrazas de hostelería de la ciudad, combinando datos abiertos, modelización urbana y visualización web interactiva.

¿Qué resuelve este proyecto?

Convierte un fenómeno urbano difícil de interpretar —la sombra real sobre las terrazas— en información medible, comparable y accesible para ciudadanía, hostelería y administración pública.

Cómo está construido

El sistema integra descarga automatizada de datos, reconstrucción geométrica de terrazas, generación de un modelo digital de superficie, simulación semanal de sombras y publicación en la nube para su consulta directa desde el navegador.

Cómo se ha creado el proyecto

El desarrollo se ha organizado como un pipeline geoespacial completo, desde la adquisición de datos hasta la visualización final. A continuación se explica el proceso de forma clara y paso a paso.

1

Descarga masiva de datos LiDAR

El primer bloque del proyecto consiste en construir un modelo físico preciso de Madrid. Para ello se emplean los datos LiDAR del PNOA, obtenidos a través del IGN, utilizando la tercera cobertura disponible para el periodo 2022–2025.

  • Se descargan todas las teselas del municipio de Madrid.
  • La descarga se automatiza mediante la herramienta oficial en Java.
  • Esto permite manejar un volumen muy grande de información de forma sistemática.
2

Generación del Modelo Digital de Superficie

A partir de los puntos LiDAR se genera un DSM o MDS de alta resolución. Cada raster representa la altura de edificios, vegetación y otros elementos urbanos.

  • Cada píxel representa una resolución de 1 metro.
  • Se crea un raster por tesela.
  • Después todas las teselas se fusionan en un modelo continuo para toda la ciudad.
3

Transformación de las terrazas administrativas

Los datos de terrazas del Ayuntamiento están originalmente en formato puntual. Por eso fue necesario transformar esos puntos en polígonos que representaran de forma realista la superficie física de cada terraza.

1
Proyección a fachada Se proyecta cada punto hacia la fachada del edificio más cercano usando cartografía catastral.
2
Dirección normal Se calcula la orientación perpendicular a la fachada para estimar la salida de la terraza hacia la acera.
3
Extensión hasta acera Se prolonga la geometría hacia la acera hasta localizar la zona de implantación posible.
4
Geometría inicial Se genera un primer polígono orientado según esa dirección calculada.
5
Ajuste por superficie real El tamaño del polígono se adapta a la superficie autorizada de la terraza.
6
Recorte final Se recorta con capas de aceras y viales para evitar invadir la calzada.
4

Enriquecimiento con más datos abiertos

Una vez obtenidas las geometrías realistas de las terrazas, se incorporan nuevas variables procedentes de otros conjuntos de datos abiertos para mejorar el análisis y la navegación en el visor.

  • Se añade la tipología del local, como restaurante, cafetería o bar.
  • Se calcula la proximidad a zonas verdes en un radio de 50 metros.
  • Esto permite activar filtros avanzados en la visualización final.
5

Simulación semanal de sombras

Con las terrazas ya modeladas y el DSM preparado, se simula la sombra incidente utilizando análisis geoespacial y cálculo de posición solar con herramientas como r.sun y r.horizon de GRASS GIS.

  • Se simula un día representativo por semana.
  • Se calculan intervalos horarios entre las 06:00 y las 22:00.
  • Se aplica un buffer adicional para captar sombras cercanas que afectan realmente a la terraza.
6

Almacenamiento en GeoParquet y publicación web

Los resultados del análisis se almacenan en formato GeoParquet, que permite consultas eficientes y un acceso optimizado desde aplicaciones web.

  • Las métricas incluyen superficie en sombra y porcentaje de cobertura.
  • Los ficheros se publican en Cloudflare R2 para su acceso directo.
  • El visor se despliega con GitHub Pages, evitando infraestructura compleja adicional.

Ejemplo técnico

El visor web

La última fase del proyecto consiste en poner toda esta información al servicio del usuario mediante una interfaz clara, pública e interactiva.

Búsqueda y selección

Permite localizar terrazas y explorar cada caso concreto sobre el mapa.

Filtro por tipo

Facilita comparar bares, cafeterías, restaurantes y otros tipos de locales.

Navegación temporal

El control horario permite ver cómo evoluciona la sombra a lo largo del día.

Consulta accesible

La información se muestra de forma comprensible también para usuarios no técnicos.

Tecnologías utilizadas: Python para el procesamiento geoespacial, GRASS GIS para la simulación de sombras, GeoParquet para el almacenamiento, Cloudflare R2 para la publicación de resultados, y Leaflet + DuckDB WebAssembly para la consulta interactiva desde navegador.

Conjuntos de datos principales

El proyecto combina varias fuentes públicas para crear un sistema coherente, reutilizable y escalable a escala de ciudad.

Base física y urbana

PNOA LiDAR 3ª cobertura (2022–2025) Base para construir el modelo digital de superficie y representar edificios, arbolado y otros elementos urbanos.
Catastro Se utiliza para identificar la fachada más cercana y orientar correctamente las terrazas generadas.
Cartografía 1:1000, aceras y viales Fundamental para el recorte geométrico, la validación espacial y el ajuste físico de las terrazas sobre el espacio público.

Contexto, búsqueda y filtrado

Terrazas del Ayuntamiento de Madrid Punto de partida administrativo para localizar y modelar cada terraza de la ciudad.
Actividades económicas Permite clasificar los locales por tipología y habilitar filtros temáticos en el visor.
Zonas verdes y callejero oficial Se emplean para medir proximidad a áreas ajardinadas y facilitar la búsqueda y navegación del usuario.

Este proyecto demuestra cómo la reutilización de datos abiertos puede transformarse en una herramienta visual, replicable y útil para comprender mejor el espacio público urbano.
